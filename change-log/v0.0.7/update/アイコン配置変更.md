# 右サイドバー(Activity Strip)のアイコン配置変更計画

## 目的
myT-xのエラーログ確認画面のアイコンを、右サイドバー（Activity Strip）の一番下に固定配置する。

## 対象ファイル
1. `myT-x/frontend/src/components/viewer/viewerRegistry.ts`
2. `myT-x/frontend/src/components/viewer/views/error-log/index.ts`
3. `myT-x/frontend/src/components/viewer/ActivityStrip.tsx`

## 実装内容詳細

### 1. `viewerRegistry.ts` の型定義の拡張
`ViewPlugin` インターフェースに、ビューの配置場所を指定できるオプションプロパティ `position` を追加する。デフォルトは上部（指定なしまたは'top'）とし、下部配置用に'bottom'の指定を可能とする。

```typescript
export interface ViewPlugin {
  id: string;
  icon: ComponentType<{ size?: number }>;
  label: string;
  component: ComponentType;
  shortcut?: string;
  position?: 'top' | 'bottom'; // 追加プロパティ
}
```

### 2. `error-log/index.ts` のビュー登録設定の変更
エラーログのビュー登録時に、追加した `position` プロパティを使って下部配置を指定する。

```typescript
registerView({
    id: "error-log",
    icon: ErrorLogIcon,
    label: "Error Log",
    component: ErrorLogView,
    shortcut: "Ctrl+Shift+L",
    position: "bottom", // 追加
});
```

### 3. `ActivityStrip.tsx` の表示レイアウトの変更
CSSのFlexbox構造（`display: flex; flex-direction: column;`）を活かし、アイコンを描画する処理（現在の`.map`）を「上部配置の要素」と「下部配置の要素」に分割して描画する。上部と下部の間には `flex: 1` などの拡張領域要素（スペーサー）を配置し、下部配置の要素が画面一番下に押し下げられるようにレイアウトを変更する。

```tsx
return (
  <div className="viewer-activity-strip">
    {/* 上部配置要素 (positionが'bottom'以外) */}
    {views.filter(v => v.position !== 'bottom').map(view => (
      <button key={view.id} ...>...</button>
    ))}

    {/* スペーサー（重要） */}
    <div style={{ flex: 1 }} />

    {/* 下部配置要素 (positionが'bottom') */}
    {views.filter(v => v.position === 'bottom').map(view => (
      <button key={view.id} ...>...</button>
    ))}
  </div>
);
```

## 期待される結果
エラーログのアイコンのみが右サイドバーの最下部に表示され、他のアイコン（ファイルツリーやDiffなど）は従来通り上部に表示されるようになる。また今後の機能追加で別のアイコンを下部に配置したくなった場合も、プロパティ一つで汎用的に対応可能となる。
