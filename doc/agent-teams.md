# Agent Teams（AI チーム開発）

myT-x の最大の特徴は、複数の AI エージェントを1つのアプリ内でチームとして連携させられることです。

---

## 概要

Agent Teams を使うと、以下のことができます：

- 各ペインに異なる AI エージェント（Claude Code、Codex CLI、Gemini CLI）を起動
- MCP オーケストレーションで、エージェント間のタスク送受信を自動化
- チーム定義を保存して、いつでも同じ構成で再起動

---

## Agent Team セッションの作り方

1. 左サイドバーの **+ 新規セッション** をクリック
2. セッション名と作業ディレクトリを入力
3. **Agent Team として開始** にチェックを入れる
4. **作成** をクリック

サイドバーで **A** バッジが付いたセッションが Agent Team セッションです。

> Agent Teams を利用するには、tmux-shim がインストールされている必要があります。
> 未インストールの場合は「(shim not installed)」と表示されます。

---

## チームの作成と管理

右サイドバーの **オーケストレーターチーム**（`Ctrl+Shift+T`）でチームを管理します。

### チーム一覧画面

| 表示 | 内容 |
|------|------|
| チーム名 | チームの名前 |
| メンバー数 | チームに所属するエージェントの数 |
| 並び替えボタン | ドラッグまたは矢印ボタンで順序変更 |

| ボタン | 説明 |
|--------|------|
| **+ 新規チーム** | 新しいチームを作成 |
| ✏ 編集 | チームの設定を変更 |
| 📋 コピー | チームを複製 |
| 🗑 削除 | チームを削除（確認あり） |
| ▶ チーム開始 | チームを起動 |

### チーム編集画面

| 項目 | 説明 |
|------|------|
| **チーム名** | チームの名前（重複不可） |
| **メンバーリスト** | チームに所属するエージェントの一覧 |

#### メンバーの操作

| ボタン | 説明 |
|--------|------|
| **+ メンバー追加** | 新しいメンバーを追加 |
| **+ 他のチームからコピー** | 既存チームのメンバーを複製して追加 |
| ✏ 編集 | メンバーの設定を変更 |
| 📋 コピー | メンバーを複製 |
| 🗑 削除 | メンバーを削除 |

### メンバー編集画面

| 項目 | 説明 |
|------|------|
| **ペイン名** | ペインに表示される名前（チーム内で一意） |
| **エージェント / ロール** | エージェントの役割や専門領域 |
| **モデル** | 使用する AI モデル（オーバーライド設定） |
| **システムプロンプト / 指示** | エージェントへの初期指示 |
| **ブートストラップ待機時間** | エージェント起動までの待機秒数 |

---

## チームの起動

1. チーム一覧画面で **▶ チーム開始** をクリック
2. 起動ダイアログでメンバー構成を確認
3. **起動** をクリック

起動すると、各メンバーにペインが自動で割り当てられ、設定された指示がブートストラップとして送信されます。

---

## オーケストレーション MCP

### 仕組み

myT-x にはオーケストレーション MCP が内蔵されています。
各ペインの AI エージェントは MCP を通じて以下のことができます：

- 他のペインにタスクを送信
- 自分宛のタスクを受信
- チームメンバーの一覧を確認

### 外部 AI からの接続方法

MCP Manager（`Ctrl+Shift+M`）で接続情報を確認できます。

```
myT-x mcp stdio --session <セッション名> --mcp <MCP-ID>
```

`--session` フラグの代わりに、環境変数 `MYTX_SESSION` でセッション名を指定することもできます。

### 対応 AI ツール

| ツール | 接続方法 |
|--------|---------|
| Claude Code | MCP stdio 接続 |
| Codex CLI | MCP stdio 接続 |
| Gemini CLI | MCP stdio 接続 |

## お試し手順

### 簡単な例

1. Agent Team セッションを作成
2. ペインを 3 つに分割（%0, %1, %2）
3. 各ペインで AI を起動
4. 最初のペインに以下を入力：

```
`orchestrator.md` を読んでください。
あなたは pane %0 です。%1, %2 が居ますので、テスターとしての役割を登録して
テストでメッセージを送信して応答を貰ってください。
```

> 軽量モデル（Haiku や Codex Low）で動作を把握してから、本番のモデルに切り替えることをおすすめします。

## ペインツールバーからのメンバー追加

各ペインのツールバーにある **👤+ メンバー追加** ボタンから、既存のチームにメンバーを追加できます。

## モデル自動切り替え

Agent Teams では、親エージェントが子エージェントを起動します。
設定画面の「Agent Model」タブで、子エージェントのモデルを自動で切り替えられます。

詳しくは [設定 > Agent Model](settings.md#タブ-4-agent-model) を参照してください。

---

## キャンバスモード

Agent Teams の連携状況を視覚的に確認するには、キャンバスモードが便利です。

切替ボタンをクリックすると、各ペインがノードとして表示され、タスクの送受信が線（エッジ）で表示されます。

| エッジの表示 | 意味 |
|------------|------|
| 合計タスク数 | 送受信されたタスクの総数 |
| pending | 未処理のタスク |
| completed | 完了したタスク |
| failed | 失敗したタスク |
| abandoned | 中断されたタスク |

## チームの保存場所

| 場所 | パス | 用途 |
|------|------|------|
| グローバル | `%LOCALAPPDATA%\myT-x\teams\` | どのプロジェクトでも使えるチーム定義 |
| プロジェクトローカル | `.myT-x\teams\`（リポジトリ内） | プロジェクト固有のチーム定義 |
