# 設定

画面左上の **設定** ボタンから設定画面を開きます。6 つのタブで構成されています。

> 一部の設定はアプリケーションの再起動後に反映されます。

---

## タブ 1: 基本設定

### Shell

ペインで起動するシェルを選択します。

| 選択肢 | 説明 |
|--------|------|
| powershell.exe | Windows PowerShell（デフォルト） |
| pwsh | PowerShell 7+ |
| cmd | コマンドプロンプト |
| bash | Git Bash など |
| WSL | Windows Subsystem for Linux |

### Prefix（プレフィックスキー）

ターミナル操作のトリガーとなるキーの組み合わせです。
デフォルトは `Ctrl+b` で、tmux と同じ操作感です。

**使い方:** プレフィックスキーを押す → ステータスバーに「プレフィックス」表示 → アクションキーを押す

### Quake Mode

ON にすると、グローバルホットキーでウィンドウの表示/非表示を切り替えられます。
他のアプリケーションで作業中でも、一瞬でターミナルを呼び出せます。

### Global Hotkey

Quake Mode のショートカットキーです。デフォルトは `Ctrl+Shift+F12`。

> 他のアプリと同じキーを設定すると動作しません。別のキーに変更してください。

### デフォルトセッションディレクトリ

クイックスタート時に使われるディレクトリを指定します。
「フォルダを選択...」ボタンで選ぶか、直接パスを入力できます。

## タブ 2: キーバインド

プレフィックスキーの後に押すアクションキーを変更できます。

| 操作 | デフォルト | 説明 |
|------|-----------|------|
| 左右分割 | `%` | ペインを左右に分割 |
| 上下分割 | `"` | ペインを上下に分割 |
| ズーム切替 | `z` | ペインを最大化/元に戻す |
| ペインを閉じる | `x` | 現在のペインを閉じる |
| デタッチ | `d` | セッションを切断（維持） |

### ビューアショートカット

各ビューアを開くためのキーボードショートカットも変更できます。

| ビューア | デフォルト |
|---------|-----------|
| Editor | `Ctrl+Shift+W` |
| File Tree | `Ctrl+Shift+E` |
| Git Graph | `Ctrl+Shift+G` |
| Diff | `Ctrl+Shift+D` |
| Input History | `Ctrl+Shift+H` |
| MCP Manager | `Ctrl+Shift+M` |
| ペインスケジューラ | `Ctrl+Shift+K` |
| タスクスケジューラ | `Ctrl+Shift+Q` |
| チーム | `Ctrl+Shift+T` |
| Error Log | `Ctrl+Shift+L` |

## タブ 3: Worktree（Git 連携）

Git リポジトリでセッションを作成すると使える機能です。

### 有効 / 無効

OFF にすると、Worktree 関連の操作がすべて非表示になります。

### 強制削除

ON にすると、未コミットの変更があっても Worktree を削除できます。
OFF の場合は確認ダイアログが表示されます。

### セットアップスクリプト

Worktree 作成後に自動実行するコマンドを登録できます。

**例:**
- `npm install` — Node.js の依存関係をインストール
- `go mod download` — Go のモジュールをダウンロード

「+ 追加」ボタンで複数のコマンドを登録できます。上から順に実行されます。

### コピーファイル

Git で管理していないファイルを、新しい Worktree に自動コピーします。

**例:** `.env`, `.env.local`

### コピーディレクトリ

フォルダごとコピーする場合はこちらに登録します。

**例:** `.vscode`

## タブ 4: Agent Model

Claude Code の Agent Teams で、子エージェントのモデルを自動で切り替える設定です。

### 一括置換

| 項目 | 説明 |
|------|------|
| **置換元 (From)** | 元のモデル名。`ALL` を指定すると全モデルが対象 |
| **置換先 (To)** | 差し替えるモデル名 |

**例:** Opus を Sonnet に切り替えてコストを削減

```
置換元: claude-opus-4-6
置換先: claude-sonnet-4-5-20250929
```

### エージェント名ごとのオーバーライド

特定のエージェントだけ別のモデルを使いたい場合に設定します。

| 項目 | 説明 |
|------|------|
| エージェント名 | 部分一致（大文字小文字を区別しない）。5 文字以上 |
| モデル | このエージェントで使用するモデル |

- 上から順に評価され、最初にマッチしたルールが使われます
- 「+ オーバーライド追加」ボタンで追加

**使い分けの例:**

| やりたいこと | 設定 |
|------------|------|
| 全体のコスト削減 | 一括で Opus → Sonnet |
| セキュリティだけ慎重に | `security` に Opus をオーバーライド |
| 用途別に最適化 | 複数のオーバーライドを使い分け |

## タブ 5: Claude Code 環境変数

Claude Code セッション作成時に自動で適用される環境変数です。

### デフォルトで有効

ON にすると、セッション作成ダイアログで「Claude Code 環境変数を使用」がデフォルトで ON になります。

### 思考レベル（CLAUDE_CODE_EFFORT_LEVEL）

| レベル | 特徴 |
|--------|------|
| **high** | じっくり考える。複雑なタスクや重要な判断に |
| **medium** | バランス型。通常の作業に |
| **low** | 素早く返答。単純な作業やチャットに |

### 変数リスト

70 種類以上の Claude Code 公式環境変数を設定できます。

| カテゴリ | 主な変数 |
|---------|---------|
| API認証 | `ANTHROPIC_API_KEY`, `ANTHROPIC_AUTH_TOKEN` |
| モデル設定 | `ANTHROPIC_MODEL`, `ANTHROPIC_DEFAULT_SONNET_MODEL` |
| 思考制御 | `MAX_THINKING_TOKENS` |
| 機能制御 | `CLAUDE_CODE_DISABLE_AUTO_MEMORY`, `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS` |
| MCP設定 | `MCP_TIMEOUT`, `MCP_TOOL_TIMEOUT` |

「+ 環境変数追加」ボタンで変数を追加、各行の ✕ ボタンで削除できます。

> `PATH` などのシステム変数は上書きできません。

---

## タブ 6: ペイン環境変数

すべてのペインに自動で適用される追加の環境変数です。

### デフォルトで有効

ON にすると、セッション作成ダイアログで「ペイン環境変数を使用」がデフォルトで ON になります。

### 変数リスト

好きな環境変数を追加できます。

**例:**
- `HTTP_PROXY` — プロキシ設定
- `GOPATH` — Go のワークスペースパス

---

## 設定ファイルの場所

設定は以下のパスに YAML 形式で保存されます：

```
%LOCALAPPDATA%\myT-x\config.yaml
```

設定ファイルに問題があってもアプリは起動します。
パースエラーはログに出力され、デフォルト設定で動作します。
