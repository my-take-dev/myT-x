# タスクスケジューラ

myT-x には 2 種類のスケジューラがあり、繰り返し作業や連続タスクを自動化できます。

---

## ペインスケジューラ（定期実行）

**ショートカット:** `Ctrl+Shift+K`

設定した間隔で、ペインにメッセージを繰り返し送信します。

### 使い道

- AI エージェントに定期的な指示を送る
- 一定間隔でコマンドを実行する
- 監視タスクの自動化

### スケジューラの作成

1. Activity Strip の **スケジューラ** アイコン（`Ctrl+Shift+K`）を開く
2. **+ 新規スケジューラ** をクリック
3. 以下を設定：

| 項目 | 説明 | 例 |
|------|------|-----|
| タイトル | スケジューラの名前 | 「定期レビュー」 |
| 対象ペイン | メッセージを送るペイン | %0 |
| メッセージ | 送信する内容 | 「進捗を報告してください」 |
| 間隔（秒） | 送信間隔 | 300（5分） |
| 最大回数 | 0 で無制限 | 10 |

4. **開始** をクリック

### スケジューラの操作

| 状態 | 表示 | 操作 |
|------|------|------|
| 実行中 | running | 停止ボタンで停止 |
| 停止 | stopped | 再開ボタンで再開 |
| 完了 | completed | 最大回数に到達 |

## タスクスケジューラ（順次実行）

**ショートカット:** `Ctrl+Shift+Q`

タスクをキューに登録し、1 つずつ順番に自動実行します。

### 使い道

- 複数のタスクを順番に実行
- AI エージェントへの段階的な指示出し
- ビルド → テスト → デプロイ のようなパイプライン

### タスクの登録

1. Activity Strip の **タスクスケジューラ** アイコン（`Ctrl+Shift+Q`）を開く
2. **+ 新規タスク** をクリック
3. 以下を設定：

| 項目 | 説明 | 例 |
|------|------|-----|
| タイトル | タスクの名前 | 「テスト実行」 |
| メッセージ | ペインに送信するコマンド | `go test ./...` |
| 対象ペイン | 送信先ペイン | %0 |
| クリアしてから実行 | 実行前にペインをクリア | ON |
| クリアコマンド | クリアに使うコマンド | `clear` |

4. **保存** で登録

### キューの操作

| ボタン | 説明 |
|--------|------|
| ▶ **キュー開始** | 登録済みタスクの順次実行を開始 |
| ⏸ **一時停止** | 現在のタスク完了後に一時停止 |
| ▶ **再開** | 一時停止したキューを再開 |
| ⏹ **停止** | キューを完全停止。未実行タスクは skipped |

### タスクの状態

| 状態 | 意味 |
|------|------|
| **pending** | 待機中（まだ実行されていない） |
| **completed** | 正常に完了 |
| **failed** | 実行に失敗 |
| **abandoned** | キュー停止により中断 |

### テンプレート

よく使うタスクの組み合わせをテンプレートとして保存できます。

| 操作 | やり方 |
|------|--------|
| 保存 | タスクを登録した後、テンプレートとして保存 |
| 読み込み | テンプレート選択で呼び出し |
| 削除 | テンプレートの削除ボタン |

---

## 2 つのスケジューラの使い分け

| | ペインスケジューラ | タスクスケジューラ |
|---|---|---|
| **実行パターン** | 同じメッセージを繰り返し | 異なるタスクを順番に |
| **用途** | 定期実行、監視 | パイプライン、段階的指示 |
| **完了条件** | 回数制限 or 手動停止 | 全タスク完了 or 手動停止 |
| **設定の複雑さ** | シンプル | タスクごとに詳細設定可 |
