# ビューアシステム

画面右端の **Activity Strip** からアクセスする 14 の開発ツールです。
アイコンをクリック、またはキーボードショートカットで開閉できます。`Escape` で閉じます。

## 表示モード

Activity Strip 下部のボタンで切り替えられます。

| モード | 説明 |
|--------|------|
| **オーバーレイ** | メインエリアの上に重ねて表示。ターミナルが隠れます |
| **ドッキング** | メインエリアの右に横並びで表示。境界線をドラッグして幅を調整できます |

---

## 1. Editor（エディタ）

**ショートカット:** `Ctrl+Shift+O`（v1.0.4 で `Ctrl+Shift+W` から変更）

Monaco Editor ベースのコードエディタです。ファイルツリーと編集エリアを横並びに表示します。

### 左パネル（ファイルツリー / 検索）

| 操作 | やり方 |
|------|--------|
| フォルダを開く/閉じる | フォルダアイコンをクリック |
| ファイルを選択 | ファイル名をクリック → 右パネルで編集 |
| ファイル検索 | ヘッダーの 🔍 ボタンで検索モードに切替 |

#### ヘッダーボタン

| ボタン | 説明 |
|--------|------|
| 🔍 | ファイル検索モードに切替 |
| +F | 新規ファイルを作成 |
| +D | 新規フォルダを作成 |
| ↻ | ファイルツリーを再読込 |

#### コンテキストメニュー（右クリック）

| 項目 | 説明 |
|------|------|
| Copy Path / Copy Relative Path | パスをコピー |
| New File / New Folder | 選択ディレクトリ下に作成 |
| Rename | 名前を変更 |
| Delete | ファイル / フォルダを削除（確認あり） |

### 右パネル（Monaco エディタ）

| 要素 | 説明 |
|------|------|
| ファイル名 + `*` | 未保存の変更があると `*` が表示される |
| 言語ラベル | 拡張子から自動検出した言語名 |
| **保存** ボタン | 変更を保存（`Ctrl+S` でも保存可） |

**注意事項:**
- 1 MB を超えるファイルは先頭部分のみ読み込まれ、ツールバーに警告が表示される
- `Ctrl+F` はMonaco Editor の検索ウィンドウを開くため、ファイル検索にはヘッダーの 🔍 を使う

## 2. File View（ファイルビュー）

**ショートカット:** `Ctrl+Shift+E`

セッションの作業ディレクトリ内のドキュメント系ファイル（v1.0.6 で `file-tree` から `file-view` に改名）をツリー表示します。
ドキュメント執筆と仕様確認に最適化されており、対象拡張子のみが Tree に表示されます。

### 表示対象ファイル（拡張子フィルタ）

| 拡張子 | プレビュー |
|------|----------|
| `.md` | Markdown レンダリング（相対パス画像対応） |
| `.mmd` | Mermaid 図 |
| `.drawio` / `.drawio.svg` / `.drawio.xml` | draw.io 図 |
| `.yaml` / `.yml` / `.json` | Swagger / OpenAPI 自動判定（先頭 1KB に `openapi:` / `swagger:` を検出時のみ Swagger UI を起動、そうでなければ Raw 表示） |
| `.db` / `.sqlite` / `.sqlite3` | SQLite Viewer（テーブル一覧 + 行データ + CSV エクスポート） |
| `.vg.json` / `.vl.json` | Vega / Vega-Lite チャート（SVG レンダリング、エクスポート・ソース表示対応） |
| `.wavedrom.json` | WaveDrom デジタル波形図 |

Markdown 内の Markmap（マインドマップ）記法にも対応しています。

子孫に対象ファイルが存在しないフォルダは Tree から自動的に隠れます。

### 左パネル（ツリー / 検索）

| 操作 | やり方 |
|------|--------|
| フォルダを開く/閉じる | フォルダアイコンをクリック |
| ファイルを選択 | ファイル名をクリック → 右パネルにプレビュー表示 |
| ファイル検索 | `Ctrl+F` で検索モードに切替。名前やパスで検索 |

### 右パネル（ファイルプレビュー）

| 種類 | 表示 |
|------|------|
| Markdown | `react-markdown` + GFM。相対パス `<img>` は `DevPanelReadBinary` 経由で blob URL 化。アウトラインパネル付き（v1.0.9） |
| Markmap | Markdown 内のマインドマップ記法を `markmap-view` でインタラクティブ表示 |
| Mermaid | `mermaid@11`（lazy load） |
| Swagger / OpenAPI | `swagger-ui-react@5`（YAML/JSON 両対応、lazy load） |
| draw.io | `.drawio.svg` は `<img>`、`.drawio` / `.drawio.xml` は XML プレビュー |
| Vega / Vega-Lite | `vega-embed` によるチャート・グラフ表示（SVG エクスポート・ソース表示対応） |
| WaveDrom | デジタル波形図レンダリング |
| SQLite | テーブル一覧 + カラム情報 + 行データ（仮想スクロール）+ ページング + CSV エクスポート |
| その他 | Raw テキスト表示（行選択・コピー対応） |

### Markdown アウトライン（v1.0.9）

`.md` ファイルを開くと、右パネル上部にアウトラインパネルが表示されます。

| 操作 | やり方 |
|------|--------|
| アウトラインを開閉 | パネル上部の折りたたみボタン |
| 見出しへジャンプ | アウトライン内の見出し名をクリック |

- 見出しに自動でアンカー ID が生成されます（重複は連番で回避）
- ページ内リンク（`#heading-anchor`）をクリックするとスムーズにスクロールします
- コンテンツ変更時はアウトラインが自動リセットされます

### ヘッダーボタン

| ボタン | 説明 |
|--------|------|
| Raw / Preview 切替 | レンダリング表示と原文表示の切替 |
| ↻ | ファイルツリーを再読込 |
| ✕ | ビューアを閉じる |

### SQLite Viewer

`.db` / `.sqlite` / `.sqlite3` ファイルを読み取り専用で開き、データを閲覧できます。

| 要素 | 説明 |
|------|------|
| 左ペイン | テーブル一覧（クリックで切替） |
| 右ペイン上部 | カラム情報（型・NULL 可否・主キー） |
| 右ペイン中央 | 行データ（仮想スクロール、NULL と空文字を区別表示） |
| ページング | 前へ / 次へボタン |
| **CSV エクスポート** | 現在のテーブルを CSV ファイルとして保存 |

接続は読み取り専用（`mode=ro&_pragma=busy_timeout(5000)`）でパスジェイル付きです。SQL インジェクション対策としてテーブル名は double-quote エスケープされます。

## 4. Diff（差分ビュー）

**ショートカット:** `Ctrl+Shift+D`

`git diff HEAD` の結果をビジュアルに表示します。ステージング操作もここから行えます。

### ヘッダー

| 要素 | 説明 |
|------|------|
| **ツリー / フラット** 切替 | 表示モードを変更 |
| ↻ | 再読込 |
| ✕ | 閉じる |

### 統計バー

変更の概要が表示されます：`+追加行数` `-削除行数` `Files: ファイル数`

### ツリーモード

**左パネル（ファイルツリー）**
- ディレクトリ構造で変更ファイルを表示
- ファイルをクリックすると差分を表示

**右パネル（コミットパネル）**
- ブランチ情報
- **コミットメッセージ** テキストエリア
- Git操作ボタン：

| ボタン | 説明 |
|--------|------|
| **Pull** | リモートから最新を取得してマージ |
| **Fetch** | リモートから最新を取得（マージなし） |
| **Commit** | ステージ済みの変更をコミット |
| **Commit & Push** | コミットしてリモートに送信 |
| **Push** | コミット済みの変更をリモートに送信 |

### フラットモード（ステージングリスト）

すべての変更ファイルを一覧表示し、ステージング操作ができます。

| 操作 | やり方 |
|------|--------|
| ファイルをステージ | ファイル行の＋ボタン |
| ファイルをアンステージ | ファイル行の－ボタン |
| 変更を破棄 | ファイル行の破棄ボタン（確認あり） |
| 全てステージ | **Stage All** ボタン |
| 全てアンステージ | **Unstage All** ボタン |

### 差分表示

| 表示 | 意味 |
|------|------|
| 緑背景 | 追加された行 |
| 赤背景 | 削除された行 |
| 灰色背景 | 変更なしの文脈行 |
| 「Expand hidden lines」 | 折りたたまれた行を展開 |

### Diff Review（インラインコメント）

Working Diff 専用のインラインコメント機能です（v1.0.9 で追加）。git-graph の Diff 画面には影響しません。

#### コメントの追加

1. Diff 行にカーソルを合わせると行の左端に `+` ボタンが表示されます
2. `+` をクリックするとその行にインラインコメント入力欄が展開されます
3. コメントを入力して「追加」（または `Ctrl+Enter`）で保存、「キャンセル」（または `Escape`）で閉じます

#### アクションバーからの送信

コメントを追加するとアクションバーに件数バッジが表示されます。

| 要素 | 説明 |
|------|------|
| 💬 Comments (N) | 保存済みコメント件数 |
| ペイン選択 | 送信先のペインを選択 |
| **送信** ボタン | 全コメントを Markdown 形式で選択ペインに一括送信（Bracketed Paste） |
| **Single Task Runnerへ登録** | コメントを Single Task Runner のタスクとして登録（Single Task Runner MCP が有効なセッションのみ表示） |

保存済みかつ未送信のコメントがある行は、Diff 上で薄いピンク系の背景でハイライトされます。
単一行コメントと複数行範囲コメントの両方に対応し、送信または Single Task Runner 登録でコメントが削除されるとハイライトも消えます。

#### Single Task Runner への登録

Single Task Runner MCP が有効なセッションでは、レビューコメントを直接送信せず、Single Task Runner のキューへ登録できます。

- コメント 1 件につき 1 タスクとして登録されます
- タスクタイトルは `レビュー指摘修正` です
- タスク本文は通常送信と同じ Markdown 形式です
- 登録後は Single Task Runner 一覧が開きます
- Start は自動実行されず、Single Task Runner 側の開始操作で実行します

#### 送信される Markdown 形式

```markdown
# Code Review Comments

## `src/utils/auth.ts` (L42)
\`\`\`
const token = getToken();
\`\`\`
> tokenがnullの場合のエラーハンドリングが不足。ログ出力を追加してください。
```

送信後はコメントがクリアされます。

## 6. MCP Manager（MCP 管理）

**ショートカット:** `Ctrl+Shift+M`

セッションで利用可能な MCP（Model Context Protocol）サーバーを管理します。

### カテゴリ

| カテゴリ | 内容 |
|---------|------|
| **Agent Orchestrator** | オーケストレーション用 MCP。エージェント間通信に使用 |
| **Single Task Runner** | シングルタスクランナー MCP。1 ペインで順次タスク実行 |
| **LSP-MCP** | 言語サーバー MCP。200 種以上の言語拡張をサポート |

### 操作

- カテゴリをクリックして切替
- 各 MCP の接続情報を確認
- MCP の有効/無効を切替

### 外部接続

MCP Manager には、外部の AI ツール（Claude Code、Codex CLI、Gemini CLI）から接続するための情報が表示されます。
`--session` フラグの使い方もここで確認できます。

### 関連ドキュメント

- [MCP オーケストレーター ツールリファレンス](mcp-orchestrator.md) — 全 19 ツールの詳細仕様

## 9. Single Task Runner（シングルタスクランナー）

**ショートカット:** `Ctrl+Shift+J`（v1.0.4 で `Ctrl+Shift+R` から変更）

オーケストレーター不要で、1 つのペインに対してタスクを順次実行する軽量タスクランナーです。MCP ツール経由でタスクのキュー登録・完了・失敗報告ができます。

### タスクリスト画面

| 表示 | 内容 |
|------|------|
| タスク名 | タスクのタイトル |
| 状態 | pending / sending / active / done / failed / cancelled |
| エラーメッセージ | 失敗時の理由 |
| クリアしてから実行 | editable なタスクは一覧上の checkbox で切替可能 |

| ボタン | 説明 |
|--------|------|
| **+ 新規タスク** | タスクをキューに追加 |
| ▶ **キュー開始** | タスクの順次実行を開始 |
| ⏹ **キュー停止** | キューを停止 |

### タスク登録画面

| 項目 | 説明 |
|------|------|
| タイトル | タスクの名前 |
| メッセージ | ペインに送信する指示 |
| **クリアしてから実行** | 実行前にペインをクリアするか |

### MCP ツール

Single Task Runner は 6 つの MCP ツールを提供します：

| ツール | 説明 |
|--------|------|
| `enqueue_task` | タスクをキューに追加 |
| `complete_task` | 現在のタスクを完了としてマーク |
| `fail_task` | 現在のタスクを失敗としてマーク |
| `list_queue` | キュー内のタスク一覧を取得 |
| `cancel_task` | タスクをキャンセル |
| `help` | ツール一覧とヘルプを表示 |

## 12. Usage Dashboard（使用状況ダッシュボード）

**ショートカット:** `Ctrl+Shift+U`

Claude Code CLI と Codex CLI の使用統計を集計・可視化するビューアです（v1.0.4 で追加）。
現在開いている myT-x セッションの作業ディレクトリをスコープとして集計します。

### ソース選択

表示モードはリスト式セレクターで切り替えます。初期表示は比較モードで、Claude Code と Codex を横並びで表示します。

| 選択 | 集計対象 |
|------|---------|
| **Claude Code** | `~/.claude/projects/**/*.jsonl` + `subagents/*.jsonl` |
| **Codex** | `.codex/sessions/**/*.jsonl` + `state_5.sqlite` |
| **Compare** | 選択したソースを横並びで比較表示 |

比較モードでは checkbox で表示ソースを選択できます。最低 1 件は選択が維持され、最大 3 件まで選択できます。
内部データ取得は常に両方のソースを対象に行うため、表示切替だけでは再集計されません。

### 表示要素

| 要素 | 内容 |
|------|------|
| 概要カード | 総セッション数、メッセージ数、ツール呼び出し数 |
| トップエージェント | 呼び出された Agent 上位 |
| トップスキル | 利用された Skill 上位 |
| トップスラッシュコマンド | 実行された SlashCommand 上位 |
| 日次アクティビティ | 過去 30 日の利用件数（recharts BarChart、v1.0.5 で追加） |
| ソースヘルスバナー | データソースの読み取りエラーや欠落の警告 |

### カテゴリセクション（v1.0.9）

エージェント / スキル / スラッシュコマンドそれぞれにカテゴリタブが追加されました。

| 要素 | 説明 |
|------|------|
| カテゴリタブ | Agent / Skill / SlashCommand をタブで切替 |
| ランキング表示 | 上位 N 件のリスト表示 |
| 日次グラフ表示 | 項目ごとの日次棒グラフ（`ItemDailyUsageChart`） |
| 単一 / 積層切替 | 棒グラフの表示モードを切替 |
| オーバーフロー集約 | 上位以外は「その他」にまとめて表示 |

### キャッシュと更新

- 集計結果は `.myT-x/usage-dashboard.json` に JSON キャッシュされ、再アクセス時の処理を高速化
- mtime ベースで差分検出するため変更のないファイルは再パースされない
- 手動更新ボタンで強制再集計
- SQLite アクセスは読み取り専用

## 13. Session Memo

**ショートカット:** `Ctrl+Shift+N`

セッション単位のメモを右サイドバーで編集できるビューアです。
アプリケーション起動中はフロントエンド側で draft を保持し、保存ボタンを押すと対象セッションの作業ディレクトリ配下へ永続化します。

### 保存先

| 項目 | 内容 |
|------|------|
| 保存ファイル | `{セッション作業ディレクトリ}\\.myT-x\\session-memo.md` |
| 保存方式 | atomic write（一時ファイル → rename） |
| サイズ上限 | 1 MiB |

### 操作

| 操作 | 説明 |
|------|------|
| メモ入力 | テキストエリアに自由に入力 |
| **保存** | 現在のメモを `session-memo.md` に保存 |
| セッション切替 | セッションごとの draft に切り替え |

セッション名を変更しても draft が失われにくいよう、セッションキー単位で管理されます。
保存済みファイルが存在しない場合は空メモとして表示されます。

## 14. Prompt Presets（プロンプトプリセット）

**ショートカット:** `Ctrl+Shift+P`

よく使うプロンプト本文を「プリセット」として登録・管理するビューアです（v1.0.6 で追加）。
チャット入力欄の **Prompt Presets** ドロップダウンから本文をテキストエリア末尾に追記できます。

### スコープ

| スコープ | 保存先 |
|---------|--------|
| **グローバル** | `%LOCALAPPDATA%\myT-x\prompt-presets.json` |
| **プロジェクト** | `{セッション作業ディレクトリ}\.myT-x\prompt-presets.json` |

プロジェクトスコープはセッション選択時のみ利用可能です。

### 操作

| 操作 | 説明 |
|------|------|
| **+ 新規** | プリセットを新規追加（名前 + 本文） |
| ✏ 編集 | プリセットの内容を更新 |
| 🗑 削除 | プリセットを削除（確認あり） |
| ↕ 並び替え | ドラッグでチャット入力ドロップダウンの表示順を変更 |

### 制約

| 項目 | 値 |
|------|-----|
| 1 スコープあたりの最大件数 | 200 件 |
| ID 形式 | UUID v4 |
| 書込み方式 | atomic write（一時ファイル → rename） |
